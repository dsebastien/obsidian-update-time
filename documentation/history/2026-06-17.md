# 2026-06-17

## Accomplished

- Implemented [issue #7](https://github.com/dsebastien/obsidian-update-time/issues/7): front-matter updates no longer kick the cursor out of Live Preview tables. Writes are now debounced until typing pauses, and skipped entirely when nothing changes.

## Root cause (issue #7)

The `modify` handler called `processFrontMatter` on every event. `processFrontMatter` always rewrites the file, refreshing the editor and (in Live Preview tables) moving the cursor out of the cell. The `MINUTES_BETWEEN_SAVES` window only gated the `updated` _value_, not whether a write happened.

## Decisions

- **Two layered debounces, kept distinct.** Invocation debounce (`saveDelayInSeconds`, idle, reset-on-keystroke) prevents mid-edit writes; value debounce (`MINUTES_BETWEEN_SAVES`) still limits `updated` churn. Did not merge them — they answer different questions (when to look vs. whether to bump the value).
- **Default delay = 2s.** Small enough that timestamps stay fresh, large enough to clear typing. User-tunable; `0` restores old immediate behavior.
- **Skip no-op writes via cached front matter probe** rather than trying to suppress `processFrontMatter`'s write (which always writes). Strictly better than before; worst case is one redundant write equal to today's behavior.
- **Helper mirrors the backfill helper** so both write paths share testable, pure logic.
