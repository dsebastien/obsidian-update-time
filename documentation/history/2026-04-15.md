# 2026-04-15 — Migration from Nx to Bun template

## Summary

Migrated `obsidian-update-time` from its Nx/npm/Jest/esbuild setup to the Bun-based Obsidian plugin template (same layout as `obsidian-plugin-template`). Plugin semver unchanged: **1.0.13**.

## What changed

### Tooling

| Before                                       | After                                                                  |
| -------------------------------------------- | ---------------------------------------------------------------------- |
| npm + `package-lock.json`                    | Bun + `bun.lock`                                                       |
| Nx workspace (`apps/plugin/`, `libs/`, etc.) | Flat `src/` layout                                                     |
| esbuild via `@nx/esbuild`                    | Bun bundler via `scripts/build.ts`                                     |
| Jest + ts-jest                               | `bun test` with `.spec.ts`                                             |
| ESLint (`.eslintrc.json`)                    | Flat config `eslint.config.ts` (typescript-eslint + obsidianmd plugin) |
| Prettier v2 (`.prettierrc`)                  | Prettier v3 (`prettier.config.cjs`)                                    |
| `version-bump.mjs`                           | `scripts/update-version.ts` + `scripts/version-bump.ts`                |
| `.github/ci.yml`, `.github/release.yml`      | `.github/workflows/ci.yml`, `release.yml`                              |
| Husky v9.0.11                                | Husky v9.1.7 + commitlint + commitizen                                 |

### Source layout

- `apps/plugin/src/` → `src/` (via `git mv` to preserve history).
- `src/assets/styles.css` → `src/styles.src.css` (Tailwind source; now `@import 'tailwindcss';` + `@apply` rules).
- Entry point unchanged: `src/main.ts` re-exports `UpdateTimePlugin` from `src/app/plugin.ts`.
- Fixed `src/app/utils/log.ts` manifest import path (`../../../../../manifest.json` → `../../../manifest.json`).
- Added `bun:test` imports to `parse-date.fn.spec.ts`.
- Copied `src/test-setup.ts` from the template (mocks `obsidian` module for tests; referenced by `bunfig.toml` via `preload`).

### Dependencies

Runtime deps preserved from the Nx `package.json`:

- `date-fns` (3.6.0) — used by `parse-date.fn.ts` and `plugin.ts`.
- `immer` (bumped to 11.0.1 to match template) — used by `plugin.ts` and `settingTab/index.ts`.
- `tslib` (2.4.0) — kept for TS helpers.

Dropped: `@popperjs/core` (not imported anywhere in the source), Nx packages, Jest, ts-jest, esbuild, nvm/npmrc pins.

### Documentation

- Replaced the Nx-adapted `AGENTS.md` with the template's content, pruned the "Building a plugin from this template" section, inserted a plugin-specific `## Project overview`, and added the vault-safety rule under Agent do/don't.
- Created `documentation/` (Architecture, Business Rules, Configuration, Domain Model) and this history file.
- Created `docs/` Jekyll site (overview, usage, configuration, tips, release notes) mirroring the template's structure.
- Created `CHANGELOG.md` seeded with the current version.
- Aligned `README.md` with the template's section order (title, pitch, features, install, configuration, known issues, contributing, license, support).

## Decisions

- Plugin id stays `update-time` (published in the community catalog since 1.0.0).
- Semver stays at **1.0.13**; no version bump was performed. The next release will be generated by `scripts/generate-changelog.ts` / `scripts/version-bump.ts`.
- `versions.json` preserved as-is (1.0.0 → 0.15.0 ... 1.0.13 → 0.15.0).
- Tailwind v4 adopted (`@tailwindcss/cli`). The single existing rule (`.support-header-margin`) was ported to `@apply mt-3`.
