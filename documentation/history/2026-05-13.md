## Decisions

- **Trim + fallback to defaults** for invalid input. Empty / whitespace-only / missing values resolve to `PROPERTY_CREATED` / `PROPERTY_UPDATED` at write time. No error UI ‚Äî keeps the settings UX simple and matches how the existing `ignoredFolders` field tolerates missing/invalid data on load.
- **No migration of existing notes** when a property name changes. The plugin only affects future writes. Documented in both `docs/configuration.md` and the per-field description in the settings tab.
- **No business-rule change required.** Invariants #2 and #3 (preserve user `created`; debounce `updated`) are about property values, not keys; they still hold when the key is renamed.
